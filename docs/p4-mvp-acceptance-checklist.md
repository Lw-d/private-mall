# P4 MVP 验收清单

记录时间：2026-05-28

本文档用于收口 P4 微信小程序 MVP，区分“已满足 P4 本地 MVP”“需要联调环境复验”和“明确延后”的事项。

## 验收结论

P4 可以进入收尾状态。当前代码已经覆盖私域商品小程序的本地 MVP 主链路：

```text
登录 -> 浏览商品 -> 选择 SKU -> 加入购物车 -> 创建订单 -> mock 支付 -> 后台发货 -> 用户确认收货
```

正式上线前仍需要在 P5 处理真实微信支付、微信登录配置、生产环境构建和联调验收。

## P5 前必须复验

在 Docker、MySQL、Redis 和 API 可用环境中执行：

```bash
pnpm db:init
pnpm --filter @mall/server build
pnpm --filter @mall/server start
pnpm smoke:transaction
```

小程序构建建议在非沙箱本机或 CI 中复验：

```bash
pnpm --filter @mall/miniapp dev:weapp
```

## 明确延后事项

| 事项                  | 延后原因                                           | 建议阶段 |
| --------------------- | -------------------------------------------------- | -------- |
| 真实微信支付          | 需要商户号、证书、回调地址和验签配置               | P5       |
| 真实微信 code2Session | 需要小程序 appId / secret 和微信接口联调           | P5       |
| 收货地址模块          | 会牵动地址表、订单地址快照、创建订单入参和后台展示 | P5/P6    |
| 优惠券 / 积分 / 运费  | 属于营销和价格规则增强，不阻塞交易主链路           | P6       |
| 售后退款              | 需要完整售后流程、退款回调和后台审核               | P6       |
| 物流轨迹              | 当前只展示物流公司和单号，轨迹订阅可独立接入       | P6       |

## 收口建议

- P4 作为本地 MVP 可以收口。
- 下一阶段建议进入 P5：联调上线准备。
- P5 优先顺序建议为：真实微信登录、真实微信支付、构建 / CI 复验、环境变量与部署文档。
