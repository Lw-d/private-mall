## 自动回归覆盖

`pnpm smoke:transaction` 当前覆盖：

- API 健康检查。
- Swagger 用户订单分页文档。
- 小程序 mock 登录。
- 后台管理员登录。
- 默认收货地址查询。
- 地址管理：
  - 新增临时地址。
  - 设置临时地址为默认。
  - 恢复原默认地址。
  - 删除临时地址。
- 从购物车创建订单时写入收货地址快照。
- 优惠券领取、锁定、取消释放、支付核销。
- 积分抵扣、积分发放、退款扣回和抵扣返还。
- 后台搜索订单。
- 后台发货。
- 后台发货后自动生成物流轨迹。
- 用户订单详情返回物流轨迹。
- 后台手动追加物流轨迹。
- 用户订单详情可见后台追加的物流轨迹。
- 后台订单列表可按 `logisticsTraceStatus=IN_TRANSIT` 筛选到当前 smoke 订单。
- 用户确认收货。
- 部分退款和全额退款。

最近一次完整回归：

```text
pnpm db:up
pnpm --filter @mall/server prisma:generate
pnpm --filter @mall/server prisma:migrate:deploy
pnpm db:seed
pnpm --filter @mall/server build
pnpm smoke:transaction
```

结果：

```text
auto logistics trace ok
manual logistics trace ok
admin logistics status filter ok
Smoke transaction passed.
```

## API 兼容说明

物流轨迹响应字段、后台手动追加轨迹和后台物流状态筛选参数见：

```text
docs/api-compatibility.md
```

关键兼容点：

- `Order.logisticsTraces` 为可选字段，旧数据或旧调用方应按空数组处理。
- 后台订单列表新增 `logisticsTraceStatus` 可选筛选参数。
- `orders` 表上的 `logisticsCompany`、`trackingNo`、`shippedAt`、`deliveryRemark` 仍保留，用于发货快照和无轨迹时的兼容展示。

## 明确延后事项

以下事项不阻塞 P7 当前阶段收口：

| 事项            | 建议阶段 | 说明                                                      |
| --------------- | -------- | --------------------------------------------------------- |
| 真实物流查询    | P8       | 当前轨迹由后台维护，未接快递 100、顺丰、菜鸟等真实接口。  |
| 物流订阅 / 回调 | P8       | 当前没有自动订阅快递轨迹或接收快递状态回调。              |
| 多包裹 / 拆单   | P8       | 当前物流轨迹按订单维度记录，不支持一个订单多个包裹。      |
| 运费模板        | P8       | 当前订单仍未接运费模板、包邮规则或偏远地区规则。          |
| 售后退货物流    | P8       | 当前退款主链路已有，未做退货物流和售后详情模型。          |
| 地址智能解析    | P8       | 当前地址手填，未接微信地址选择、OCR 或智能省市区解析。    |
| 地址区域编码    | P8       | 当前保存省市区文本，未保存行政区划编码。                  |
| 物流操作审计    | P8       | 当前记录轨迹内容，未记录操作人、来源和变更审计。          |
| E2E 视觉回归    | P8       | 当前主要依赖 build 和 smoke，未接 Playwright / 真机截图。 |
